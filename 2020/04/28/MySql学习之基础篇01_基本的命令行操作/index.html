数据库操作

一、数据库查询语句分类

名称 解释 命令
DDL（数据定义语言），Define 定义和管理数据对象，如数据库，数据表等 CREATE、DROP、ALTER
DML（数据操作语言），Manager 用于操作数据库对象中所包含的数据 INSERT、UPDATE、DELETE
DQL（数据查询语言），Query 用于查询数据库数据 SELECT
DCL（数据控制语言），Control 用于管理数据库的语言，包括管理权限及数据更改 GRANT、commit、rollback

二、基本的命令行操作

命令行连接

mysql -uroot -p123456        --连接数据库

update mysql.user set authentication_string=password('123456') where user='root' and Host = 'localhost';         --修改用户密码

flush privileges;            --刷新权限

-------------------------------------------------------
-- 所有语句都要用分号结尾

show databases;                --查看所有的数据库

use school;                    --使用/切换到名为school的数据库

show tables;                --查看当前使用的数据库中所有的表

describe student;            --显示数据库中所有表的信息

create database abc01;        --创建一个名为abc01的数据库

exit;                        --退出连接

三、创建数据库

DDL的一种

create table [if not exists] `表名`(
   '字段名1' 列类型 [属性][索引][注释],
   '字段名2' 列类型 [属性][索引][注释],
  #...
   '字段名n' 列类型 [属性][索引][注释]
)[表类型][表字符集][注释];

说明 : 反引号用于区别MySQL保留字与普通字符而引入的 (键盘esc下面的键).

四、操作数据库

操作数据库 -> 操作数据库中的表 -> 操作数据库中表的数据

1、sql中的数据值

列类型 : 规定数据库中该列存放的数据类型

数据类型

字符串

日期和时间型数值类型

NULL值

  • 理解为 “没有值” 或 “未知值”
  • 不要用NULL进行算术运算 , 结果仍为NULL

2、数据字段属性

UnSigned

  • 无符号的
  • 声明该数据列不允许负数 .

ZEROFILL

  • 0填充的
  • 不足位数的用0来填充 , 如int(3),5则为005

Auto_InCrement

  • 自动增长的 , 每添加一条数据 , 自动在上一个记录数上加 1(默认)

  • 通常用于设置主键 , 且为整数类型

  • 可定义起始值和步长

    • 当前表设置步长(AUTO_INCREMENT=100) : 只影响当前表
    • SET @@auto_increment_increment=5 ; 影响所有使用自增的表(全局)

NULL 和 NOT NULL

  • 默认为NULL , 即没有插入该列的数值
  • 如果设置为NOT NULL , 则该列必须有值

DEFAULT

  • 默认的
  • 用于设置默认值
  • 例如,性别字段,默认为”男” , 否则为 “女” ; 若无指定该列的值 , 则默认值为”男”的值

SHOW CREATE DATABASE school;

SHOW CREATE TABLE student;
-- 显示表结构
